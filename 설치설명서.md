# Discord Bridge 설치 설명서

SillyTavern(ST) 캐릭터를 **디스코드에서 평범한 채팅처럼** 대화하게 해주는 봇.

이 문서 하나로 **아무것도 없는 상태 → 작동**까지 끝낸다.
순서대로만 따라가면 된다.

```
[준비물 확인] → PART A (서버 설치) → PART B (디스코드 봇) → PART C (연결·실행)
```

---

## 0. 이게 어떻게 도는지 (30초 요약)

```
디스코드 채널 ──메시지──▶ discord-bridge (서버에서 도는 봇)
                             │ ST 폴더를 직접 읽음 (캐릭터/API키/로어북/페르소나)
                             │ AI에게 응답 생성 요청
                             ▼
디스코드 채널 ◀─캐릭터 이름·얼굴로 답장─┘
```

- 봇은 **ST와 같은 서버**에서 돌아야 한다 (ST 폴더를 직접 읽기 때문).
- 설정은 **ST 설정 패널의 🤖 Discord Bridge** 화면에서 한다 (config 파일 직접 편집 안 해도 됨).

---

## 준비물

- 서버 1대 (예: 오라클 클라우드 Ubuntu). ST와 봇이 같이 산다.
- 디스코드 계정.
- AI API 키 (ST에 이미 넣어 쓰던 거 그대로 사용 — 따로 준비 불필요).

---

# PART A. 서버에 설치

> 이미 ST가 깔려 있고 잘 쓰는 중이면 **A-1, A-2는 건너뛰고 A-3부터** 하면 된다.

## A-1. Node.js 설치 (없으면)

Ubuntu 기준:

```bash
curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash -
sudo apt-get install -y nodejs
node -v        # v20.x 나오면 OK
```

## A-2. SillyTavern 설치 (없으면)

```bash
cd ~
git clone https://github.com/SillyTavern/SillyTavern.git
cd SillyTavern
./start.sh     # 처음 실행 시 자동으로 설치됨. Ctrl+C로 끄고 다음 단계 진행
```

- ST를 외부에서 접속하려면 `config.yaml`에서 `listen: true` + 사용자 인증(`basicAuth` 등) 설정.
- 캐릭터/API 키/페르소나는 평소처럼 ST에서 세팅해 두면 봇이 그대로 읽는다.

## A-3. discord-bridge 설치

```bash
cd ~
git clone <이 저장소 주소> discord-bridge   # 또는 폴더 업로드
cd discord-bridge
npm install
```

> 설정은 나중에 ST 화면에서 한다. 지금 config.json 건드릴 필요 없음.

## A-4. ST 서버 플러그인 설치

플러그인 파일 1개를 ST의 `plugins/` 폴더에 넣는다. **하위 폴더 안 만들어도 된다 — 파일만 직접 넣으면 됨.**

```bash
cp ~/discord-bridge/st-plugin/index.js \
   ~/SillyTavern/plugins/discord-bridge-config.js
```

ST `config.yaml`에서 서버 플러그인을 켠다:

```yaml
enableServerPlugins: true
```

봇 폴더 위치가 `~/discord-bridge`가 아니면, ST 실행 시 환경변수로 알려준다:

```bash
DISCORD_BRIDGE_PATH=/실제/경로/discord-bridge ./start.sh
```

## A-5. ST 확장 설치

확장 파일들을 ST 확장 폴더에 넣는다:

```bash
mkdir -p ~/SillyTavern/data/default-user/extensions/discord-bridge
cp ~/discord-bridge/st-extension/manifest.json \
   ~/discord-bridge/st-extension/index.js \
   ~/discord-bridge/st-extension/style.css \
   ~/SillyTavern/data/default-user/extensions/discord-bridge/
```

이제 ST를 재시작한다. 콘솔에 `[discord-bridge-config] 플러그인 로드됨` 이 뜨면 성공.
브라우저로 ST 접속 → Extensions 패널에 **🤖 Discord Bridge** 가 생긴다.

# PART B. 디스코드 봇 만들기 (처음이면 여기부터)

## B-1. 내 서버 만들기

디스코드 왼쪽 `+` → "직접 만들기" → "나와 친구용". 봇이 들어올 집이다.
대화용 채널도 만들어 둔다 (예: `snow-dm`, `blaise-dm` — 캐릭터마다 1개 권장).

## B-2. 봇 생성 & 토큰 받기

1. https://discord.com/developers/applications → **New Application** → 이름 입력 → 생성
2. 왼쪽 **Bot** 탭 → **Reset Token** → 뜨는 토큰을 **복사** (이게 봇 토큰. 한 번만 보여줌)
3. 같은 화면 **Privileged Gateway Intents** → **MESSAGE CONTENT INTENT** 를 **켠다** → 저장
   - ⚠ 이거 안 켜면 봇이 메시지를 못 읽어서 **무반응**이 된다. 제일 흔한 실수.

## B-3. 봇을 내 서버에 초대

1. 왼쪽 **OAuth2** → **URL Generator**
2. **SCOPES**: `bot` 체크
3. **BOT PERMISSIONS**: `View Channels`, `Send Messages`, `Read Message History`, **`Manage Webhooks`** 체크
   - `Manage Webhooks` 가 있어야 캐릭터 이름·얼굴로 답장한다.
4. 맨 아래 생성된 URL 복사 → 브라우저에 붙여 → 내 서버 선택 → 승인
5. 서버에 봇이 보이면 OK (아직 오프라인 — 정상)

# PART C. 연결하고 실행

## C-1. ST 화면에서 설정

ST → Extensions → **🤖 Discord Bridge** 열기:

1. **Discord 봇 토큰** 칸에 B-2에서 복사한 토큰 붙여넣고 → **저장**
   (저장하면 봇 토큰으로 채널 목록을 자동으로 가져온다)
2. **커넥션 프로필** 드롭다운에서 AI 백엔드 선택
3. **채널 ↔ 캐릭터 매핑** → [+ 추가] → 채널 고르고(드롭다운) 캐릭터 고르기(드롭다운). 채널마다 반복.
4. 언어 / 한도 확인 → **저장**

## C-2. 봇 실행

config는 봇이 켜질 때 읽으므로, **저장 후 봇을 (재)시작**해야 적용된다. pm2 권장:

```bash
cd ~/discord-bridge
npm install -g pm2          # 처음 한 번만
pm2 start src/index.js --name discord-bridge
pm2 save
pm2 startup                 # 서버 재부팅 시 자동 시작
```

이후 설정을 바꾸면 재시작만:

```bash
pm2 restart discord-bridge
```

잠시 뒤 디스코드에서 봇이 **온라인**, ST 화면 상태바가 🟢 로 바뀐다.
매핑한 채널에 메시지를 보내면 캐릭터가 답한다.

# 업데이트할 때

```bash
cd ~/discord-bridge && git pull && npm install
cp st-plugin/index.js ~/SillyTavern/plugins/discord-bridge-config.js
cp st-extension/* ~/SillyTavern/data/default-user/extensions/discord-bridge/
pm2 restart discord-bridge
# ST도 재시작 (플러그인/확장 갱신 반영)
```

---

# ⚠ 배포할 때 토큰 주의

- **git 저장소에는 토큰이 없다** (config.json은 플레이스홀더). 저장소를 공유하면 안전.
- 하지만 **서버에서 돌던 폴더를 통째로 복사**해서 주면, 그 안 `config.json`에 진짜 토큰이 들어있다.
  → 배포 전 `config.json`의 `discordToken` 값을 비우거나, git 저장소 버전을 줘라.
- `.gitignore`가 `data/`(대화기록·heartbeat)를 제외하므로 커밋에는 안 섞인다.

# 문제 해결

| 증상 | 원인 / 해결 |
|---|---|
| 봇이 메시지에 무반응 | **MESSAGE CONTENT INTENT** 안 켬 (B-2) / 채널 매핑 안 함 |
| 캐릭터 이름이 아니라 봇 이름으로 답함 | **Manage Webhooks** 권한 누락 (B-3) |
| ST 화면에 🤖 Discord Bridge 없음 | 확장 폴더 위치 틀림 (A-5) / ST 재시작 안 함 |
| 상태바 "플러그인 연결 실패" | `enableServerPlugins: true` 안 함 / 플러그인 파일 위치 틀림 (A-4) |
| 채널 드롭다운이 비어있음 | 토큰 저장 안 함 / 봇을 서버에 초대 안 함 |
| 상태바 🔴 | 봇 프로세스 정지 → `pm2 restart discord-bridge` |
| `config.json을 찾을 수 없습니다` | `DISCORD_BRIDGE_PATH` 환경변수로 봇 폴더 경로 지정 (A-4) |

# 파일 구조 참고

| 위치 | 역할 |
|---|---|
| `src/` | 봇 본체 (24시간 도는 Node 프로세스) |
| `st-plugin/index.js` | ST 서버 플러그인 → `plugins/discord-bridge-config.js` 로 복사 |
| `st-extension/` | ST 확장 (설정 화면) |
| `config.json` | 봇 설정 (ST 화면에서 자동 기록 — 직접 편집 불필요) |
| `data/` | 대화 기록·상태 파일 (자동 생성, 건드리지 말 것) |
